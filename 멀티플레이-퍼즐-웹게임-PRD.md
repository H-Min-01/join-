## 1. 개요

### 한 줄 정의
앱 설치나 회원가입 없이 링크 공유만으로 친구들과 즉시 즐길 수 있는 실시간 멀티플레이 퍼즐 웹 게임.

### 제품 목표
사용자들이 앱 설치나 복잡한 회원가입 없이 링크 공유만으로 친구들과 쉽고 빠르게 실시간 멀티플레이 퍼즐 게임을 즐길 수 있는 환경을 제공합니다. 디스코드 액티비티와 유사하게 가볍고 즐거운 소셜 인터랙션을 통해 친구들과 함께하는 즐거움을 극대화하는 것이 목표입니다.

### 배경
최근 모바일 환경에서 친구들과 가볍게 즐길 수 있는 실시간 멀티플레이 게임에 대한 수요가 증가하고 있으나, 대부분의 게임은 앱 설치나 회원가입 등 참여를 위한 번거로운 절차를 요구합니다. 이로 인해 즉흥적인 게임 플레이에 진입 장벽이 존재합니다.

## 2. 핵심 가치

### 사용자 문제
사용자들은 친구들과 실시간으로 함께 즐길 수 있는 게임을 원하지만, 앱 설치, 회원가입, 복잡한 초대 과정 때문에 즉흥적이고 간편한 게임 플레이에 어려움을 겪고 있습니다. 디스코드 액티비티처럼 손쉽게 참여하고 즐길 수 있는 소셜 게임 경험에 대한 니즈가 있습니다.

### 해결 방식
링크 공유만으로 즉시 참여 가능한 멀티플레이 웹 게임 플랫폼을 제공합니다. 별도의 앱 설치나 회원가입 없이 게스트 ID를 통해 누구나 손쉽게 방에 입장하여 Wordle 또는 숫자야구 게임을 실시간으로 즐길 수 있도록 합니다.

### 차별점
- 설치 없이 즉시 플레이 가능한 웹 기반 서비스로 접근 장벽이 낮습니다.
- 카카오톡 등 메신저 링크 공유 중심으로 친구 초대 흐름이 자연스럽습니다.
- 디스코드 액티비티와 유사한 가벼운 멀티플레이 경험을 제공합니다.
- 소규모 친구 그룹 중심의 캐주얼한 게임 환경에 최적화되어 있습니다.

## 3. 타겟 및 시나리오

### 타겟 사용자
별도의 앱 설치나 복잡한 절차 없이 친구들과 즉석에서 가벼운 게임을 즐기고자 하는 소규모 친구 그룹 사용자. 주 사용 환경은 모바일 메신저이며, 연령대는 10대 후반부터 30대 초반까지를 우선 타겟으로 설정합니다.

### 대표 사용 시나리오
1. 사용자 A가 게임 웹사이트에 접속해 `방 생성`을 누릅니다.
2. 게임 종류, 공개 여부, 글자 수 또는 자릿수를 선택해 방을 생성합니다.
3. 시스템은 고유한 룸 ID가 포함된 링크를 생성하고 대기실을 보여줍니다.
4. 사용자 A는 해당 링크를 카카오톡 채팅방에 공유합니다.
5. 사용자 B와 C는 링크를 클릭해 로그인 없이 게스트로 즉시 입장합니다.
6. 세 명의 사용자는 동일한 게임 화면에서 각자 추측을 입력하고 제출합니다.
7. 각 참여자는 서로의 실제 입력값이 아닌 진행 상태를 실시간으로 확인합니다.
8. 게임 종료 후 결과를 확인하고 다시 하거나 결과를 공유합니다.

### 유저 플로우 요약
- 홈/진입: 메인 홈, 방 생성, 방 코드 입력
- 방 생성: 게임 종류 선택, 공개 여부 설정, 옵션 설정, 방 생성 완료
- 대기실/공유: 참여자 목록 확인, 링크 복사, 카카오톡 공유, 기타 메신저 공유, 게임 시작
- 게임 플레이: Wordle 또는 숫자야구 진행, 실시간 진행 현황 확인, 추측 제출
- 게임 결과: 승리/패배 확인, 다시 하기, 결과 공유, 홈 복귀

## 4. 성공 지표

| 지표 | 목표 | 정의 | 측정 방식 |
| --- | --- | --- | --- |
| 주간 활성 사용자(WAU) | MVP 2주 내 20명 | 7일 동안 1회 이상 방 생성, 방 입장, 추측 제출을 수행한 유니크 게스트 ID 수 | `guest_id` 기준 유니크 사용자 집계 |
| 게임 완료율 | MVP 2주 내 60% | 방 입장 또는 게임 시작 대비 실제 게임 종료 비율 | `game_complete ÷ room_join` 또는 `game_complete ÷ game_start` |
| 초대 성공률 | MVP 2주 내 50% | 공유된 링크가 실제 입장으로 이어진 비율 | `room_join_from_link ÷ share_click` |
| 7일 재방문율 | MVP 2주 내 30% | 첫 방문 이후 7일 내 재활동한 사용자 비율 | 첫 `guest_id` 생성 코호트 기준 재방문 집계 |

## 5. 속성 설정

### 제품 속성

| 항목 | 설정값 |
| --- | --- |
| 카테고리 | 엔터테인먼트 / 미디어 |
| 사용자 역할 | 일반 사용자 |
| 지원 디바이스 | 모바일 웹 우선, 데스크톱 웹 지원 |
| 참여 방식 | 공유 링크 입장, 룸 ID 직접 입력 |
| 인증 방식 | 비회원 게스트 ID 기반 세션 참여 |
| 핵심 게임 모드 | Wordle, 숫자야구 |
| 방 공개 범위 | `Public`, `Invite Only` |
| 방 식별 방식 | 고유 룸 ID 자동 생성 |
| 공유 채널 | 링크 복사, 카카오톡 공유, Web Share 기반 기타 메신저 공유 |
| 실시간 통신 | WebSocket 기반 양방향 동기화 |
| 상태 저장 | 서버 DB 저장 + 세션 복구 |

### 게임 옵션 기본값

| 게임 | 기본값 | 선택 범위 |
| --- | --- | --- |
| Wordle | 5글자 | 4글자 ~ 7글자 |
| 숫자야구 | 4자리 | 3자리 ~ 5자리 |

### 상태 관리 원칙
- 실제 입력값은 다른 플레이어에게 노출하지 않습니다.
- 다른 플레이어에게는 시도 횟수, 완료 여부, 진행 상태만 노출합니다.
- 페이지 새로고침 또는 일시 이탈 후 재접속 시 기존 상태를 복구해야 합니다.
- 서버 상태를 단일 진실 공급원으로 두고 클라이언트는 이를 우선 반영합니다.

## 7. 주요 요구사항

### 7.1 방 생성 및 설정
- 사용자는 게임 종류, 공개 여부, 옵션값을 선택할 수 있어야 합니다.
- 방 생성 시 시스템은 충돌 없는 고유 룸 ID를 생성해야 합니다.
- 생성된 방의 설정 정보와 공유 링크를 즉시 확인할 수 있어야 합니다.

### 7.2 방 입장
- 사용자는 공유 링크 클릭만으로 즉시 방에 입장할 수 있어야 합니다.
- 사용자는 룸 ID 입력만으로도 방에 입장할 수 있어야 합니다.
- 별도 회원가입 없이 게스트 ID가 자동 생성되어야 합니다.
- 유효하지 않거나 만료된 방에 대해서는 명확한 오류 안내가 제공되어야 합니다.

### 7.3 게임 진행
- Wordle은 글자 위치 및 포함 여부를 정확히 판정해야 합니다.
- 숫자야구는 스트라이크와 볼 결과를 정확히 계산해야 합니다.
- 모든 참여자는 동일한 게임 상태를 공유해야 합니다.
- 다른 참여자의 실제 입력값은 숨기되 진행도는 실시간으로 보여야 합니다.

### 7.4 실시간 동기화 및 영속성
- 게임 상태 변경은 모든 참여자에게 즉시 반영되어야 합니다.
- 네트워크 지연 또는 재연결 상황에서도 게임 상태 일관성이 유지되어야 합니다.
- 새로고침 또는 재접속 후에도 정답, 시도 기록, 참여자 목록이 복원되어야 합니다.

### 7.5 공유
- 방 생성 후 링크 복사 버튼이 항상 보여야 합니다.
- 카카오톡 공유는 모바일과 데스크톱 환경 모두에서 동작해야 합니다.
- 공유 성공 여부에 대한 사용자 피드백이 제공되어야 합니다.

## 8. 비기능 요구사항

### 성능
- 대기실 입장과 게임 재진입은 사용자가 지연을 체감하지 않는 수준으로 동작해야 합니다.
- 실시간 상태 변경은 짧은 지연 내에 모든 참여자에게 반영되어야 합니다.

### 신뢰성
- WebSocket 연결이 끊겨도 자동 재연결을 시도해야 합니다.
- 상태 저장 실패 시 재시도 또는 복구 로직이 필요합니다.
- 동시 접속 및 동시 입력 시 서버 기준으로 상태 충돌을 해결해야 합니다.

### 접근성
- 복사 및 공유 버튼은 스크린 리더와 키보드 탐색을 지원해야 합니다.
- 모바일 터치와 데스크톱 마우스 모두에서 자연스럽게 조작 가능해야 합니다.

### 보안 및 개인정보
- 게스트 ID는 예측이 어려운 식별자로 생성해야 합니다.
- 세션 정보는 브라우저 저장소와 서버 세션 정책에 따라 안전하게 관리해야 합니다.
- 방 접근 제어는 공개 방과 초대 전용 방 정책에 맞게 동작해야 합니다.

## 10. 출시 판단 기준

- 사용자가 1분 이내에 방 생성부터 공유까지 완료할 수 있어야 합니다.
- 초대받은 사용자가 로그인 없이 10초 내 대기실에 진입할 수 있어야 합니다.
- 3인 기준 멀티플레이에서 게임 상태 동기화가 안정적으로 유지되어야 합니다.
- 새로고침 또는 재입장 시 진행 중 게임을 복구할 수 있어야 합니다.
- 최소 한 개 메신저 채널에서 공유 후 실제 입장 전환이 정상 측정되어야 합니다.
